# 하루1소재 프로젝트 규칙

## 절대 규칙

- **`.env` 파일은 절대 커밋하지 않는다.** `.env`, `.env.local`, `.env.*.local` 등 환경변수 파일은 어떤 경우에도 git에 추가하지 않는다.
- **git 커밋은 사용자가 명시적으로 요청할 때만 한다.** 임의로 커밋하지 않는다.

## 코드 수정 전 확인 필수

코드를 수정하기 전에 반드시 사용자에게:
1. **어떤 작업을 할 것인지** (변경 대상 파일, 변경 내용 요약)
2. **어떤 결과가 나오는지** (예상 동작 변화)

를 먼저 설명하고 확인을 받은 후 진행한다.

단, 사용자가 명확하게 "해줘"라고 지시한 단순 작업(예: 특정 텍스트 변경, 색상 변경 등)은 바로 진행해도 된다.

## 파일 편집 안전 규칙

- `page.tsx`처럼 큰 파일은 Edit 툴 대신 반드시 Python `data.replace(old, new, 1)` 방식으로 수정한다.
- 수정 전 항상 `assert old in data`로 패턴 존재 여부를 확인한다.
- 수정 후 항상 `npx tsc --noEmit`으로 타입 에러를 확인한다.

## 기술 스택

- Next.js App Router, TypeScript, Tailwind CSS v4
- 라우팅: `app/[[...slug]]/page.tsx` 단일 catch-all 라우트
